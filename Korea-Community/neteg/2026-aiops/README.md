### Abstract
인프라/네트워크 엔지니어 관점에서, 터미널 에이전트(Claude Code)와 함께 GKE 위에 쿠버네티스 인프라를 구성·배포한 실전 경험을 공유합니다. 자연어 프롬프트로 매니페스트를 생성하고 ArgoCD로 배포하며, 그 과정이 Git 저장소에 그대로 쌓여 살아있는 운영 표준이 되는 **GitAIOps** 흐름을 라이브 데모로 보여줍니다.

### Key Topics
| # | Topic | Description |
|---|-------|-------------|
| 1 | **과거 → 현재** | 인프라 엔지니어의 일하는 방식 변화 (장비 앞 CLI 직접 입력 → 자연어 요청으로 매니페스트 생성) |
| 2 | **클라우드 → 쿠버네티스** | 멀티클라우드의 복잡성을 하나의 언어로 추상화하는 "클라우드 OS"로서의 쿠버네티스 |
| 3 | **AI 도구의 발전 단계** | 대화형 → 코드 보조 → 터미널 에이전트 → 자율 에이전트, 지금은 터미널 에이전트의 시대 |
| 4 | **실전 구성 배포** | GKE 위에서 ch2~ch8까지 (ArgoCD + GitHub Actions, 관측 가능성, 무중단 배포, 멀티테넌시) |
| 5 | **Live Demo** | "CronJob 만들어줘" — 기존 컨텍스트 파악부터 매니페스트 생성·ArgoCD 동기화까지 |
| 6 | **GitAIOps** | 구성 배포가 저장소에 쌓여(CLAUDE.md·JOURNEY.md·ADR·매니페스트) 즉석에서 온보딩 문서·Runbook이 만들어지고, 돌아보니 살아있는 운영 표준이 되어 있다 |

### 함께 보면 좋은 책
**AI 시대에 개발자가 알아야 하는 인프라 구성 배포 with 클로드 코드** — 이 발표의 실습 흐름(ch2~ch9)을 책으로 정리했습니다.
